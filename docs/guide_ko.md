# 블로그 검증 및 사용 가이드

이 문서는 구축된 멀티 토픽 블로그를 검증하고 사용하는 방법을 설명합니다.

## 1. 프로젝트 실행

터미널에서 다음 명령어를 입력하여 로컬 개발 서버를 실행하세요.

```bash
npm run dev
```

서버가 실행되면 브라우저에서 `http://localhost:3000`으로 접속합니다.

## 2. 화면 구성 및 디자인 테마 확인

이 블로그는 주제(Topic)별로 다른 테마가 적용되어 있습니다.

### 메인 페이지
- **주소**: `http://localhost:3000`
- **확인할 점**: 4개의 주제(Tech, Law, Faith, Education) 카드가 보이고, 각 카드의 색상과 인터랙션이 잘 작동하는지 확인하세요.

### 주제별 페이지 (Theming 검증)

각 주제를 클릭하여 스타일이 변경되는지 확인합니다.

1. **Tech (테크)**
   - **주소**: `/tech`
   - **테마**: 다크 모드 (검은 배경, 흰 글씨), 모노스페이스(코딩) 폰트.
2. **Law (법률)**
   - **주소**: `/law`
   - **테마**: 라이트 모드 (따뜻한 아이보리 배경), 세리프(명조) 폰트 (신뢰감).
3. **Faith (신앙)**
   - **주소**: `/faith`
   - **테마**: 차분한 톤, 산세리프 폰트.
4. **Education (교육)**
   - **주소**: `/education`
   - **테마**: 밝은 화이트 톤.

## 3. 기능 및 콘텐츠 확인 (MDX)

작성된 샘플 글을 통해 특수 기능을 확인하세요.

### Tech: 코드 하이라이팅 (Shiki)
- **확인 경로**: `/tech/hello-world`
- **검증**: 글 내부의 TypeScript 코드 블록이 알록달록하게 문법 강조(Syntax Highlighting)가 되어 있는지 확인하세요.

### Law: 각주 (Footnotes)
- **확인 경로**: `/law/legal-note`
- **검증**: 텍스트 중 `[^1]` 같은 각주 링크가 있고, 클릭 시 문서 하단의 설명으로 이동하는지 확인하세요.

## 4. 글 작성하기 (Keystatic CMS)

비개발자도 쉽게 글을 쓸 수 있는 관리자 페이지입니다.

1. **관리자 페이지 접속**: `http://localhost:3000/keystatic`
2. **글쓰기**:
   - 원하는 주제(Collection)를 선택합니다 (예: Tech).
   - "Create" 버튼을 누릅니다.
   - 제목(Title), 날짜, 내용을 입력합니다.
   - **Tech** 글이라면 코드 블록을, **Law** 글이라면 각주를 넣어보세요.
   - 상단의 "Save" 버튼을 누르거나 `Command + S`를 누릅니다.
3. **파일 생성 확인**:
   - 프로젝트 폴더의 `content/tech/` (또는 해당 주제 폴더) 안에 새로운 `.mdx` 파일이 생성되었는지 확인합니다.
4. **브라우저 확인**:
   - 개발 서버(localhost:3000)에서 해당 주제 페이지로 이동하여 새 글이 목록에 뜨는지 확인합니다.

## 5. 트러블슈팅

- **스타일이 깨져 보인다면?**: Tailwind CSS v4가 정상적으로 빌드되지 않았을 수 있습니다. 서버를 껐다 켜보세요.
- **새 글이 안 보인다면?**: 페이지를 새로고침하세요. (Next.js 캐시 때문일 수 있습니다)
